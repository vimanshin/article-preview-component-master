## Правила SCSS

### Модульность
- Файлы бьём на партиалы (`_variables.scss`, `_mixins.scss`) и подключаем
  через `@use`, а НЕ `@import` — `@import` в Sass устарел.
- При `@use` обращаемся к переменным и миксинам через namespace:
  `variables.$color-primary`, `mixins.tablet()` — не напрямую
- Никогда не используй `@use ... as *`

### BEM и вложенность
- BEM через `&`: `&__element`, `&--modifier`:

```scss
.card {
  &__title { … }   // → .card__title
  &--active { … }  // → .card--active
}
```

- Вложенность — максимум 2–3 уровня. Глубже — выноси в отдельный блок
- Не стилизуй по `#id` — только по `.class`
- Классы в kebab-case

### Mobile-first
- Сначала базовые стили (для мобильных) — без медиа-запросов
- Расширения через `@include mixins.tablet()` и `@include mixins.desktop()`
- Никогда не используй `max-width` там, где можно обойтись `min-width`
- Повторяющийся брейкпоинт выносим в mixin, чтобы не дублировать число:

```scss
@mixin tablet { @media (min-width: 768px) { @content; } }
.card { padding: 8px; @include tablet { padding: 16px; } }
```

### Переменные
- Переменную заводи ТОЛЬКО если значение: повторяется В НЕСКОЛЬКИХ местах
  ИЛИ несёт смысл (цвет бренда, брейкпоинт, базовый шрифт)
- Разовые и самоочевидные значения — хардкодом, без переменных
- Не придумывай новые переменные — сначала проверь, есть ли уже в `_variables.scss`
- Правило: хорошая переменная отвечает «что это значит» (`$color-primary`), а не просто «какое тут значение»
- `$`-переменные SCSS — для статичных токенов (считаются при сборке). CSS-переменные `--var` — когда значение должно меняться в браузере (темы, тёмный режим). По умолчанию пока — `$`

### Прочее
- `!important` — только в крайней необходимости, обязательно с комментарием ПОЧЕМУ
- Не дублируй свойства без причины
- Сокращай там, где уместно: `margin: 0 auto` вместо двух отдельных свойств
- Порядок свойств пиши согласно правилам в `.postcss-sorting.json`

## Доступность (Accessibility)

### Фокус
- Никогда не пиши `outline: none` / `outline: 0` без замены на видимый кастомный фокус.
  Стандартная замена: `outline: 2px solid currentColor; outline-offset: 2px`
- Предпочитай `:focus-visible` вместо `:focus` — браузер сам решит, когда показывать
  кольцо (клавиатура — да, мышь — нет):

```scss
.button {
  &:focus-visible {
    outline: 2px solid variables.$color-primary;
    outline-offset: 2px;
  }
}
```

### Анимации и движение
- Всегда оборачивай анимации и переходы в `prefers-reduced-motion`:

```scss
.card {
  transition: transform 0.3s ease;

  @media (prefers-reduced-motion: reduce) {
    transition: none;
  }
}
```

### Скрытый контент для скринридеров
- Для «визуально скрытого, но доступного» текста используй mixin `visually-hidden`,
  а не `display: none` (оно скрывает и от скринридеров тоже):

```scss
// в _mixins.scss
@mixin visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

### Цвет и контраст
- Не передавай смысл только через цвет — дублируй иконкой, текстом или паттерном
- Минимальный контраст текста по WCAG AA: **4.5:1** для обычного текста, **3:1** для крупного (18px+)
- Если переменные цветов определяешь — проверяй пары «текст / фон» в DevTools или Figma

### Размеры интерактивных элементов
- Минимальная зона нажатия: **44×44px** (WCAG 2.5.5).
  Если элемент визуально меньше — увеличивай зону через `padding` или `min-width/min-height`
